- 2026-07-12: fixed the `nmr_any` Monte Carlo ÔåÆ add-model **crash** ÔÇö `DataClassPrivate`'s copy constructors shared the `m_info` (`DataClassPrivateObject`) pointer while the destructor deletes it, so every `Override*Table()`ÔåÆ`d.detach()` deep-copy (one per Monte Carlo worker clone, freed via `deleteLater` on the GUI event loop) deleted the *live project's* `m_info`; being a `QPointer` it silently went null and the next model's `LoadSystemParameter()` then crashed at `emit Info()->SystemParameterLoaded()` (a null read on the zero page, pinpointed with AddressSanitizer after guarding SupraFit's own SIGSEGV handler out of ASan builds). - 2026-07-11: speciation solver 12-27x faster ÔÇö replaced the BFGS inverse-Hessian update in `BFGSConcentrationSolver` with a **damped Levenberg-Marquardt Newton** method using the analytic Hessian (`diag(s_i) + ╬ú c_j m_j m_jßÁÇ`) of the convex log-space potential; steps solve `(H+╬╗I)d=-g` (Cholesky), accepted on an objective- OR gradient-norm decrease so the mass-balance residual is driven to machine precision (~1e-12) even where the objective is flat, with ╬╗ self-tuning (robust across weak/strong/competitive binding ÔÇö no fragile step cap). Also precomputes `log(╬▓)` and hoists the active-index maps out of the per-call `Objective`. Result on the new `src/tests/benchmark_speciation` harness (32-point sweeps): 1:1 2.33 msÔåÆ86 ┬Ás (27x; the old solver stalled at 7e-4 residual on host-excess points), 2:1/1:1 12x, self-aggregation 19x, 3-component 20x ÔÇö from 54-146 iterations down to a uniform 7-8, all converging to 1e-12. All model/solver tests and the NMR reference regression pass unchanged.

Legacy `MaxA/MaxB/MaxSelfA/Species` remain a backward-compatible fallback (empty `Reactions` ÔçÆ classic grid; NMR reference projects re-fit unchanged); the old nmr_any BFGS/Polynomial "Concentration solver" option and `EqnConc_2x` fallback were retired (the BFGS engine is now the single solver).
