# AI-assisted Changelog

One line per significant AI-assisted improvement (newest first).

- 2026-07-13: **warm-started speciation solver** (~3.4× faster inner solves) — `BFGSConcentrationSolver::setTotalConcentrations` no longer discards the previous solve's free concentrations, so each point of a titration sweep warm-starts from the last (the class was designed for this but the fit path cleared it every point); newly-active components fall back to the cold heuristic. Iterations ~7.5→4.0, per-solve time ~4.3µs→1.3µs at -O3, identical accuracy (strictly convex ⇒ same minimum). New `test_speciation_warmstart` verifies a threaded Monte Carlo on nmr_any stays correct with both speciation methods (per-clone engine, no warm-start race); `test_varpro_cv` (CV/RA) still green. Fits ~2.7× faster (test_varpro 9.3s→3.5s).
- 2026-07-13: **guard Monte Carlo against an unset `threads` property** — `MonteCarloStatistics::GenerateData` divided `MaxSteps / maxthreads` where `maxthreads` came from an app property the GUI/CLI set but a headless/test context does not (0 ⇒ SIGFPE); clamp to ≥1.

- 2026-07-13: **`roadmap/joint_solver.md`** — design/evaluation of a unified concentration+fit-parameter solver; found the speciation warm-start is discarded in the fit path (`setTotalConcentrations` clears `m_has_guess`), so the "7-8 iter/solve" figure is cold-start; recommends warm-started nested first, then analytic outer Jacobian via the implicit-function theorem, deferring a full-space KKT/SQP.
- 2026-07-13: **GUI switch for the speciation solver** (LevMar/Newton vs. legacy BFGS) — new `SpeciationSolver` optimizer-config key (`global.h`); `AbstractModel::setOptimizerConfig` made virtual + `UsesSpeciationEngine()` hook; `AbstractTitrationModel` (nmr_any/uvvis_any/fl_any) and `itc_any` push the method into their embedded `SpeciationEngine` on config change and at `DefineModel`; a "Speciation solver" submenu in `ModelWidget`'s Fit menu (greyed off-engine, re-fits on select), and `OptimizerWidget::Config()` round-trips the key so Apply no longer drops it. MC/CV/RA inherit it via `Clone()`.
- 2026-07-13: **fixed the red `uv_vis 2:1/1:1` VarPro equivalence test** — `{2.4,4.2}` (copied from the NMR row) is a bad-basin instance for the Beer-Lambert objective (weakly-populated A2B ⇒ flat K21 valley both solvers miss; VarPro collapses to K11≈0). The projection is correct (seeded at those constants VarPro reaches SSE~0); switched the row to the well-conditioned `{3.0,4.5}` so both solvers converge and the strong equivalence gate holds. `test_varpro` 13/0.

- 2026-07-12: fixed the `nmr_any` Monte Carlo → add-model **crash** — `DataClassPrivate`'s copy constructors shared the `m_info` (`DataClassPrivateObject`) pointer while the destructor deletes it, so every `Override*Table()`→`d.detach()` deep-copy (one per Monte Carlo worker clone, freed via `deleteLater` on the GUI event loop) deleted the *live project's* `m_info`; being a `QPointer` it silently went null and the next model's `LoadSystemParameter()` then crashed at `emit Info()->SystemParameterLoaded()` (a null read on the zero page, pinpointed with AddressSanitizer after guarding SupraFit's own SIGSEGV handler out of ASan builds). Fix: each private creates its own `m_info`. Regression `test_dataclass::testInfoOwnershipAcrossDetach` fails without the fix.
- 2026-07-12: fixed `DataTable` leaks in `DataClassPrivate` — `~DataClassPrivate` never freed the two raw tables (`m_*_raw_model`, allocated by every ctor/copy-ctor) and the four `set*Table` setters overwrote their `QPointer` without freeing the replaced table; both now free correctly (LeakSanitizer-verified, ASan-clean, `test_dataclass` 39/39). The remaining `Override*Table` leak is left for the flagged MC/table-ownership restructuring (those tables are `m_ptr_table`-tracked and freeing them from a worker thread would race the main thread).
- 2026-07-12: fixed model line/error series being hidden until manually toggled — `ModelElement::DisableSignal` passed a plain 0/1 to `LineSeries::showLine(int)`, which treats its argument as a `Qt::CheckState` and only shows on `== Qt::Checked (2)`, so the load path (`toggleActive`/`Update`/reactive binding) hid the fit + error line-series; now passes a `bool` (`state != 0`) so the correct `showLine(bool)` overload runs and they show on load.

- 2026-07-11: speciation solver 12-27x faster — replaced the BFGS inverse-Hessian update in `BFGSConcentrationSolver` with a **damped Levenberg-Marquardt Newton** method using the analytic Hessian (`diag(s_i) + Σ c_j m_j m_jᵀ`) of the convex log-space potential; steps solve `(H+λI)d=-g` (Cholesky), accepted on an objective- OR gradient-norm decrease so the mass-balance residual is driven to machine precision (~1e-12) even where the objective is flat, with λ self-tuning (robust across weak/strong/competitive binding — no fragile step cap). Also precomputes `log(β)` and hoists the active-index maps out of the per-call `Objective`. Result on the new `src/tests/benchmark_speciation` harness (32-point sweeps): 1:1 2.33 ms→86 µs (27x; the old solver stalled at 7e-4 residual on host-excess points), 2:1/1:1 12x, self-aggregation 19x, 3-component 20x — from 54-146 iterations down to a uniform 7-8, all converging to 1e-12. All model/solver tests and the NMR reference regression pass unchanged.

- 2026-07-11: N-component equilibrium models + reaction-equation editor — generalised the `*_any` titration models from the fixed 2-component host/guest system to **arbitrary N components** through a shared speciation layer: new `src/core/reactionparser.{h,cpp}` (parses free-text equations "A + B <=> AB", "2 A <=> A2", "A + C <=> AC" into an ordered component list + stoichiometry matrix; intermediate complexes may be reused as reactants for stepwise binding, e.g. "A + AB <=> A2B", which is resolved down to the elementary components, with cycle detection) and `src/core/speciationengine.{h,cpp}` (reaction system + `BFGSConcentrationSolver`, per-datapoint solve). `AbstractTitrationModel` gained `m_component_count`, `InitialConcentration(i,c)` and a dynamic `InputParameterSize()`; `nmr_any` (observed-component mole-fraction × shift) and `uvvis_any` (Beer–Lambert over free components + species × ε) now consume the engine, `itc_any` uses it for arbitrary species (stays 2-component — totals come from the cell/syringe protocol). Legacy `MaxA/MaxB/MaxSelfA/Species` remain a backward-compatible fallback (empty `Reactions` ⇒ classic grid; NMR reference projects re-fit unchanged); the old nmr_any BFGS/Polynomial "Concentration solver" option and `EqnConc_2x` fallback were retired (the BFGS engine is now the single solver).
